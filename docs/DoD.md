# Definition of Done (v1)

Para que una TARJETA/TAREA esté “Done”:
- Commit asociado con mensaje descriptivo y push realizado.
- Evidencia visual/documental guardada en /docs/evidencias/.
- Prueba manual o automatizada pasada, con resultado anotado en la tarjeta.
- Cumple los criterios de aceptación (CA) definidos para la historia de usuario correspondiente.
- Artefactos actualizados (README, endpoints, DER, modelo o documentación del sprint).
- Checklist completo al 100 % en Trello y movimiento a columna Done.

Para que un SPRINT esté “Done”:
- Todas las tarjetas comprometidas se encuentran en Done o replanificadas con justificación.
- Demo interna (video o capturas agrupadas en /docs/evidencias/).
- Retro corta registrada (3 bullets: seguir / parar / probar) dentro de /docs/evidencias/retro_sprintX.md.
- Reporte del proyecto actualizado, marcando las secciones modificadas.
- Métrica rápida: HU completadas vs planificadas o screenshot del burndown en Trello.
